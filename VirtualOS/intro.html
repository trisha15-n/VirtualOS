Page Replacement

Page Replacement is the technique of swapping out pages from physical memory when no more free frames are available, in order to make room for other pages that need to be loaded into memory.

  • If a process tries to access a page not present in physical memory, it causes a page fault, which triggers the need to load that page into memory.
  • If no free frames are available:
    • We find an occupied frame that is not currently in use.
    • The contents of the frame are written to swap space.
    • Page tables and other records are updated to indicate the page is no longer in memory.
    • The page causing the fault is loaded into the freed frame.

FIFO Page Replacement

The First-In, First-Out (FIFO) page replacement algorithm is the simplest of its kind. When a page replacement needs to happen, the oldest page in memory is chosen to be replaced.

  • We track the order in which pages enter memory using a FIFO queue.
  • When replacement occurs, the page at the head of the queue is swapped out.
  • A new page is added at the tail of the queue when loaded into memory.
